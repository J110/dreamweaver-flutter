================================================================================
DESIGN PATTERNS USED
================================================================================

1. Service Locator Pattern
   - Services designed as singletons for app-wide access
   - Each service manages its own lifecycle

2. Dependency Injection
   - Constructor-based initialization
   - Configuration parameters passed at creation

3. Stream-based Reactivity
   - Audio position/state/buffering streams
   - Auth state changes stream
   - Enables reactive UI updates

4. Interceptor Pattern
   - Dio interceptors for auth injection
   - Global error handling
   - Request/response logging

5. Repository Pattern
   - Services act as data repositories
   - Hide implementation details from consumers
   - Consistent interface across services

6. Exception Handling
   - Domain-specific exceptions
   - User-friendly error messages
   - Proper error propagation

USAGE PATTERNS
================================================================================

All services follow consistent patterns:

INITIALIZATION:
  1. Create service instance: final service = ServiceClass()
  2. Initialize: await service.initialize()
  3. Use methods: await service.methodName(params)
  4. Cleanup: await service.dispose() (where applicable)

ERROR HANDLING:
  try {
    final result = await service.method();
  } on ServiceException catch (e) {
    print(e.message);
  }

STREAM LISTENING:
  stream.listen((value) {
    // Handle updates
  });

ASYNC/AWAIT:
